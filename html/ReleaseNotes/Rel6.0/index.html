New database file format
This is probably the most significant change in this release. It introduces a large number of new capabilities in BRLCAD. The significant features this introduces include:
Machine-independent data
The geometry in the new file format is stored in an architecture-independent fashion. This removes the need to convert the geometry file to ascii representation and back to binary when moving between machines of different architectures (eg. between Big-Endian/Little-Endian machines)
Unlimited Length Object Names
Names of objects in the geometry database were previously limited to 16 characters. This restriction was burdensome when constructing complex systems. The new database format (and MGED/rt) allows for unlimited length names for objects. The only limits remaining are usability/human factors (you may wish to thing twice before using 1024-character names for all your objects).
Object Attributes
All geometry file objects can now carry arbitrary text data. This is stored as a series of name/value pairs. For example, it is possible to store an attribute called "modeler" which could indicate the name and organization of the person who created that portion of the geometry. Another attribute might be "density" whose value would be the density of the material from which the object was constructed. Analysis codes can store application specific data in these attributes and retrieve them at run-time. MGED has a GUI control panel for creating and editing object attributes.
Opaque Binary Objects
It is now possible to store arbitrary data in a BRLCAD geometry database object. For example, the entire description from an external CAD system could be stored for reference. Several features of BRLCAD geometry which require ancillary data can now use these database objects as a data source in addition to being able to use external disk files.
Reduced Disk space requirements
The new file format is much more agressive about space utilization. Many primitives now occupy sigificantly less disk space.
New dbupgrade utility for easy migration
The dbupgrade command has been created to bring old databases up to the most current version of the file format.
Dual Format capability
The BRLCAD utilities retain the ability to work with files in the previous format. Users can upgrade geometry to the new format when they choose. However, new features implemented in the geometry file may not be available under the old format.
Hidden Objects
It is now possible to mark objects in the database as "hidden". These will not be listed in ordinary displays of database content. This allows users to keep objects such as Opaque Binary Objects from cluttering listings
Deprecation of obsolete "hf" primitive
The dsp primitive (Displacement Map) is functionally equivalent and offers substantial performance advantages. The bot primitive (Bag 'o Triangles) is a likewise a substantial improvement over the existing "poly" primitve. The dbupgrade utility converts existing hf and poly primitives into dsp and bot primitives, respectively. Both hf and poly primitives are deprecated.
Introduction of 2D "sketch" primitive
The sketch primitive consists of straight lines, circular arcs, and bezier curves. There is a drawing utility in MGED for creating them.
Introduction of "extrude" primitive
The extrude primitive supports linear extrusion of closed-boundary sketch primitive drawings to create 3D objects.
New ASCII file format for new binary file format
This new ASCII format is based upon the Tcl scripting language. It allows more convenient processing and manipulation of the ASCII format.
MGED Geometry Editor Improvements
In addition to supporting both the old and new geometry file formats, there have been numerous improvements. This include:
New User's Manual
A completely updated tutorial introduction to MGED and modeling with BRLCAD. Delivered in PDF format, suitable for both online browsing and printing. Includes a complete command reference. Look for it in the mged menus at:
"Help->Manual->PDF MGED Tutorial"or
"/usr/brlcad/html/manuals/mged/VolumeII-Introduction_to_MGED.pdf"
Accelerators for complex shape construction
These allow user to automatically generate more complex primitives from existing ones: rcc-blend, rcc-cap, rcc-tgc, rpp-arch, rpp-cap, rpp-arch, tor-rcc, sph-part
Improved menu and interface layout
Abort on Raytrace Control Panel
Users Now have the option to abort a running ray-trace process right from the control panel.
Extended Primitive Type-in Support
Several Primitive types can now be created from the command prompt. These include:
  • kinematics control (grip)
  • and arbitrary regular
  • polyhedron (arbn)
Extended Primitive Auto-Generation
Several primitive types can now be auto-generated using the make command from within MGED. These include:
  • rectangular parallelopipeds (rpp)
  • arbitrary faceted (ars)
  • and arbitrary regular polyhedron (arbn)
Primitives created with this command take their initial parameters from the view center and size.
New or improved commands
The following new commands have been implemented:
  • E Extended options for creating evaluated wireframes
  • adjust: Alter attributes of an object.
  • attr: Assign or retrieve a named text attribute from an object
  • autoview: Set view size and center so that all displayed solids are in view
  • bot_decimate: Perform facet reduction on BOT by edge collapse
  • bot_face_sort Optimize a BOT primitive for faster raytracing.
  • copyeval: Copy and evaluated/transformed primitive
  • hide/unhide: mark database object as hidden/visible
  • lt: Like ls, but displays its output in a Tcl-friendly format
  • nmg_collapse: Perform facet reduction on NMG by edge collapse
  • nmg_simplify: Attempt to convert NMG to CSG primitive (ARB, TGC)
  • pathlist Lists all existing paths that start from the specified combinations and end at a primitive
  • rtedge Command line interface to the rtedge program for generating line drawings
  • shells Separates an NMG primitive into many objects, 1 object per NMG shell.
  • showmats List transformation matricies on a path
  • summary Provide database statistics
  • xpush An extended push that attempts to avoid degenerate conditions in matrix propagation.
Build Pattern Tool
This tool facilitates the construction of replicated objects in various rectangular grid and circular patterns. For example: lines of rivet heads, bolt holes, railroad ties, tire tread patterns.

Summary

[back]
  • New database file format and migration tools.
    • New file format is machine-independent (no need for g2asc | asc2g when moving between Big-Endian and Little-Endian machines).
    • New file format stores many objects using less disk space.
    • New "dbupgrade" utility for migrating old databases to new file format.
    • MGED dual-format capability (edits either old or new file formats.
    • New "opaque binary objects" in database for carrying arbitrary data
    • Object attributes: store arbitrary name/value pairs with any database object
    • Hidden geometry support.
    • Texture shader and dsp primitive can access binary objects as their data source in addition to external files
    • Tcl-based ascii database file format for new binary file format.
    • Improved terrain performance (dbupgrade converts slower hf primitives to dsp primitive)
    • Improved performance when ray-tracing bot primitives (rt_pieces)
    • Improved performance when ray-tracing dsp primitive (octtree based cell traversal)
  • A 2D "sketch" primitive which consists of straight lines, circular arcs, and bezier curves.
  • An "extrusion" primitive which allows linear extrusion of closed planar 2D sketch primitives into 3D shapes.
  • Improved overall raytrace performance
  • Ascii support for extruded bitmap (ebm) and volume (vol) primitives
  • General bug fixes, security enhancements, and speed improvements
  • Multi-device Geometry Editor (MGED) improvements
    • New "accelerators" to automatically generate more complex primitives from existing ones: rcc-blend, rcc-cap, rcc-tgc, rpp-arch, rpp-cap, rpp-arch, tor-rcc, sph-part
    • Improved menu and interface layout
    • New "Introduction to MGED" user manual
    • 'in' command support for kinematics control (grip), sketch extrusion (extrude), arbitrary regular polyhedron (arbn), and binary object (binunif) primitives
    • 'make' command support for rectangular parallelopipeds (rpp), arbitrary faceted (ars), and arbitrary regular polyhedron (arbn) primitives
    • New commands include: adjust, autoview, attr, attr_rm, copyeval, hide, lt, nmg_collapse, nmg_simplify, pathlist, shells, showmats, summary, unhide, version, xpush, rtedge, E, bot_face_sort
    • Bezier curves are interactive display support
    • New "Build Pattern Tool" for duplicating geometry in a pattern
    • New attribute editor panel
  • Supported architectures include Linux, FreeBSD, SGI IRIX, Sun, and now MacOS X
  • Improved Pro-Engineer conversion support (new converters)
    • Oriented and non-oriented triangle support
    • Unlimited name length support
  • Fastgen 4 conversion now supports character names longer than 16 characters
  • Improved and updated support documentation (man pages, command help, and documentation)
  • Improved raytracer edge detection performance (rtedge)
  • Initial support for multiply represented objects (MROs) (application library)
  • New "flat" and "invisible" shaders

New Utilities

[back]
  • pixcmp

    Although not technically new (formerly included in source distributions), pixcmp is now installed

  • rtedge

    Raytraces models, detects edges, and writes BRL-CAD color image (.pix) files

  • g_diff

    Compares two geometry database files and reports differences between the two

  • dbupgrade

    Upgrades an old BRL-CAD database to the current BRL-CAD release file format. This allows access to new functionality only available with the newer database formats. (e.g. Convert a database created with BRL-CAD rel4.5 to one that is fully functional in BRL-CAD rel6.0)

New Libraries

[back]
  • New general purpose bourne-shell-based utility library (part of regression suite)

    Includes support for file locking, semaphores, stdout/stderr/file logging, mail, cvs, packaging, and remote host querying facilities

Developer Notes

[back]
  • Compile-time optimizations that significantly increase raytrace performance (at cost of run-time debugging)
  • Removed dependency on X (and Tk) during source compilation
  • Consolidated cake build options
  • Mged 'db' developer command facilities
  • New and improved regression test suite (still in transition)
  • Improved ANSI source conformance (still in transition)
